# PM Bot 交易方案（BTC 5M / 15M Up-or-Down）

## 1. 目标

为 Polymarket 的 BTC 快周期涨跌盘构建一个**长期正期望**的交易机器人，重点覆盖：

- `btc-updown-5m-*`
- `btc-updown-15m-*`

机器人不是“每局都下注”，而是：

1. 发现当前活跃的 5M / 15M 盘
2. 拉取 Binance 实时价格与 Polymarket / Chainlink 参考数据
3. 只在存在明确 edge 时下注
4. 严格控制仓位、时段和单日回撤

## 2. 盘口规则

这类市场的结算规则：

- 如果 **结算时价格 >= 起始时价格**，则 `Up` 赢
- 否则 `Down` 赢
- 结算源：**Chainlink BTC/USD data stream**
- 注意：**不是 Binance 现货价格**，而是 Chainlink 数据

因此，系统的核心不是猜 K 线，而是交易以下几类 edge：

- 短周期动量延续
- 极端波动后的均值回归
- Binance 与 Chainlink / Polymarket 的时间差
- 重大事件窗口的方向偏移

## 3. 长期盈利原则

### 3.1 不做“每局必打”

长期稳定盈利的前提不是提高下注次数，而是**减少无 edge 下注**。

必须默认：

- 无信号 = 不下注
- 不清楚 = 不下注
- spread 太大 = 不下注
- 剩余时间太短 = 不下注

### 3.2 先过滤，再下注

交易流程：

1. **时间过滤**
2. **市场过滤**
3. **信号判断**
4. **仓位计算**
5. **执行下单**
6. **结算复盘**

## 4. 时间过滤（什么时候打）

### 4.1 高优先级时段

优先交易以下时段：

- 美股开盘前后：`09:30-11:30 ET`
- 美股尾盘前后：`15:30-16:00 ET`
- 亚盘波动切换：`19:00-21:00 ET`
- 重大宏观事件前后：
  - CPI
  - 非农
  - FOMC / Fed 决议
  - ETF 资金流 / 重大监管消息

### 4.2 低优先级 / 跳过时段

默认跳过：

- 极低波动、窄幅横盘时段
- 盘口剩余时间不够时
- 波动率不足、价格离均线过近、方向不明确时

## 5. 市场过滤（打哪一局）

### 5.1 基本过滤条件

仅交易满足以下条件的盘：

- `active = true`
- `closed = false`
- 剩余时间充足：
  - 5M：剩余至少 `>= 120s`
  - 15M：剩余至少 `>= 300s`
- 盘口点差可接受：
  - 默认 `spread <= 0.03`
- 流动性足够：
  - 5M：`liquidity >= 8_000`
  - 15M：`liquidity >= 15_000`

### 5.2 盘口价格过滤

不追极端价：

- `Up` 或 `Down` 一侧价格若已经 `> 0.62`，默认跳过
- 若价格在 `0.48 ~ 0.55` 区间，更适合做 edge 交易

原因：

- 极端价意味着市场已经大幅定价
- 高价追单，赔率差，不利于长期期望

## 6. 信号体系（下注哪个方向）

系统按优先级从高到低判断信号。若高优先级命中，则低优先级不再触发。

---

## 6.1 信号 A：Oracle 延迟 / 价格跳变跟随（最高优先级）

### 逻辑

如果 Binance 现货价格在极短时间内快速跳动，而 Polymarket / Chainlink 参考尚未完全反映，则可以顺着现货方向下注。

### 触发条件（建议初版）

- 观察窗口：最近 `10s ~ 30s`
- Binance 最新价格相对短时基准偏移：
  - `abs(delta_usd) >= 35 ~ 50`
  - 或 `abs(delta_bps) >= 4 ~ 6 bps`
- 同时短时成交量放大 / 波动放大
- 当前盘口价格仍接近 `0.50`
- 剩余时间足够

### 下注方向

- Binance 快速上冲 → 买 `Up`
- Binance 快速下杀 → 买 `Down`

### 原因

这不是赌趋势，是赌**短时信息未完全传导**。

---

## 6.2 信号 B：短周期动量延续

### 逻辑

短周期在特定时段存在弱动量延续；若方向明确，可顺势下注。

### 5M 建议条件

- 最近 3 根 1m K 线同向
- 当前价在 `EMA8` 同侧
- 1m / 3m return 同向
- RSI 未极端背离

### 15M 建议条件

- 最近 2 根 5m K 线同向
- 当前价在 `EMA8` 同侧
- `EMA8 > EMA21` 时偏多
- `EMA8 < EMA21` 时偏空

### 下注方向

- 动量向上 → 买 `Up`
- 动量向下 → 买 `Down`

### 使用限制

- 只在高波动时段使用
- 若价格已经远离均线，不追

---

## 6.3 信号 C：极端后的均值回归（主要用于 15M）

### 逻辑

15M 相比 5M 噪声更少，短时过冲后存在回归机会。

### 条件

- 价格触及 / 接近布林带外沿（如 2σ）
- RSI 过热 / 过冷
- 最近一段波动放大后开始衰减
- 没有重大新闻正在驱动单边趋势

### 下注方向

- 过度下跌、出现支撑 → 买 `Up`
- 过度上涨、出现受阻 → 买 `Down`

### 限制

- 若事件驱动极强，则不要逆势做回归

---

## 6.4 信号 D：事件窗口交易

### 逻辑

重大事件前后，方向偏移往往不对称。

### 条件

- 当前处于预设事件窗口内
- 方向判断来自外部信号：
  - 宏观预期差
  - ETF 资金流
  - 重大新闻解析

### 下注方向

- 根据事件方向偏置下注

### 限制

- 初版可先做“事件白名单 + 人工开关”
- 暂不建议完全自动化解读新闻后直接下单

## 7. 不下注条件（必须硬编码）

以下任一命中都必须跳过：

- spread 太大
- 流动性太差
- 剩余时间太短
- 当前价格偏离 0.50 太多
- 波动率太低
- 价格方向冲突（多个信号互相打架）
- 连续亏损达到暂停阈值
- 当日回撤达到风控阈值

## 8. 风控

### 8.1 单笔风险

建议初版：

- 普通信号：`1% ~ 2%` 资金/笔
- 强信号（如 oracle delay）：`3% ~ 5%` 资金/笔

### 8.2 连续亏损熔断

- 连亏 `3` 笔：暂停 `30` 分钟
- 连亏 `5` 笔：当日只允许人工恢复

### 8.3 单日最大回撤

- 当日净值回撤达到 `5%`：停止当日自动交易

### 8.4 总原则

- 活下来比抓到某一笔更重要
- 长期盈利依赖稳定执行，而非重仓豪赌

## 9. 系统实现建议（MVP）

初版建议做成 Python 项目，包含：

### 9.1 数据层

- `BinanceMarketDataClient`
  - WebSocket / REST 获取 BTC 价格与 K 线
- `PolymarketMarketClient`
  - 获取当前 5M / 15M 活跃盘
  - 读取盘口价格、流动性、时间
- `ChainlinkReferenceClient`
  - 读取参考价 / 对比偏差（若可用）

### 9.2 信号层

- `SignalEngine`
  - `oracle_delay_signal()`
  - `momentum_signal()`
  - `mean_reversion_signal()`
  - `event_signal()`
  - `decide()` 返回最终方向 / 是否下注

### 9.3 风控层

- `RiskManager`
  - 单笔仓位
  - 连亏熔断
  - 单日回撤熔断
  - no-trade 过滤器

### 9.4 执行层

- `PaperExecutor`（MVP 默认）
  - 先做 paper trading / 信号记录
- `PolymarketExecutor`（接口预留）
  - 下单接口先留 stub，不要求初版真实下单

### 9.5 记录与复盘

- trade log
- signal log
- skipped reason log
- end-of-day summary

## 10. MVP 范围（第一版真正要做的）

第一版不要过度设计，只做：

1. Python 项目脚手架
2. 配置文件
3. 拉取当前活跃 5M / 15M 盘
4. 拉取 Binance BTC 实时价格 / 简化 K 线
5. 实现三类核心信号：
   - oracle delay
   - momentum
   - mean reversion
6. 实现 no-trade 过滤器
7. 实现 risk manager
8. 支持 paper trading
9. CLI 启动与日志输出
10. 基础单元测试

## 11. 先不做的内容

以下不纳入初版：

- 完整真实下单签名与钱包交易
- 自动新闻 NLP 决策
- 分布式部署
- 复杂数据库
- UI 面板
- 多币种扩展

## 12. 成功标准

初版完成时，应满足：

1. 能发现当前活跃的 BTC 5M / 15M 盘
2. 能基于 Binance 数据生成方向信号
3. 能输出：
   - 是否下注
   - 下注方向（Up/Down）
   - 原因
   - 信号置信度
   - 建议仓位
4. 能记录 paper trade
5. 能在本地跑通测试
6. 代码结构清晰，便于后续接真实下单

## 13. 最后的纪律

长期盈利不是靠“预测神准”，而是靠以下纪律组合：

- 少打
- 只打有 edge 的时刻
- 不追极端价
- 严格限仓
- 连亏就停
- 回撤到线就停

一句话总结：

> **不是每一局都下注，而是在最有利的少数局面下注。**
